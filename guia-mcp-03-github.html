🐙 Sección 1

¿Qué puede hacer GitHub MCP?

GitHub MCP es el plugin oficial de GitHub para Claude Code. Le da acceso completo a tus repositorios: puede leer código, crear archivos, gestionar issues, revisar pull requests y mucho más — todo sin abrir el navegador.

📁

Leer y crear archivos

Accede al código de cualquier repositorio (propio o público), crea archivos nuevos, hace commits y gestiona branches.

🐛

Gestionar issues

Crea, actualiza, comenta y cierra issues. Categoriza bugs automáticamente y asígnalos al equipo correcto.

🔀

Revisar Pull Requests

Analiza el código de un PR completo, sugiere mejoras, detecta bugs y genera comentarios de revisión.

🔍

Buscar en el código

Busca patrones, funciones o lógica específica en cualquier repositorio público de GitHub.

🚀

Releases y changelogs

Genera notas de versión automáticamente a partir de los commits y PRs mergeados desde el último release.

📊

GitHub Projects

Accede a milestones, etiquetas y proyectos. Consulta el estado del roadmap sin salir del terminal.

🔑 Sección 2

Crear tu Personal Access Token de GitHub

Necesitas un token para que Claude pueda autenticarse con GitHub. Se genera en 2 minutos desde la configuración de tu cuenta.

⚠️

Seguridad importante: Tu token es como una contraseña. Nunca lo compartas, nunca lo subas a GitHub (ni siquiera en un archivo .env). Guárdalo en un lugar seguro como un gestor de contraseñas. Si se compromete, revócalo inmediatamente desde GitHub Settings.

1

Ve a GitHub Settings

Entra a tu cuenta de GitHub → haz clic en tu foto de perfil (arriba a la derecha) → Settings.

2

Accede a Developer Settings

En el menú lateral izquierdo, baja hasta el final y haz clic en Developer settings.

3

Crea un token clásico

Haz clic en Personal access tokens → Tokens (classic) → Generate new token (classic). Ponle un nombre descriptivo como "Claude MCP".

4

Selecciona los permisos necesarios

Para uso general con Claude, marca: repo (acceso completo a repos privados) y read:org. Si solo usas repos públicos, marca solo public_repo.

5

Copia y guarda el token

GitHub te muestra el token una sola vez. Cópialo y guárdalo en un lugar seguro antes de cerrar la página.

⚡ Sección 3

Instalación y configuración

Con tu token listo, la instalación es un único comando en tu terminal.

💻 Comando de instalación
claude mcp add github npx @github/github-mcp-server
# Claude te pedirá el GITHUB_PERSONAL_ACCESS_TOKEN durante la configuración

Si prefieres configurarlo manualmente, aquí está el JSON completo para agregar a tu configuración de Claude Code:

⚙️ Configuración JSON (alternativa)
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["@github/github-mcp-server"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "tu_token_aqui"
      }
    }
  }
}
✅

Verifica que funciona: Reinicia Claude Code y escríbele: "Dime cuáles son mis repositorios más recientes en GitHub". Si te lista los repos, está configurado correctamente.

🚀 Sección 4

Flujos de trabajo reales con ejemplos

Estas son las combinaciones de prompts más útiles para tu trabajo diario con GitHub MCP activo.

🔍 Code Review automático
"Revisa el PR #42 del repositorio [mi-repo]. Identifica posibles bugs, problemas de seguridad y sugiere mejoras. Sé específico y directo."
🐛 Gestión de issues
"Revisa los issues abiertos de [mi-repo] y categorízalos por: bug crítico, mejora de UX, deuda técnica o nueva funcionalidad. Lista los 3 más urgentes."
📝 Generar changelog
"Analiza todos los commits y PRs mergeados en [mi-repo] desde el tag v1.2.0 hasta hoy. Genera unas notas de versión claras para los usuarios."
📖 Documentación automática
"Lee el archivo [src/api/routes.js] de [mi-repo] y genera un README.md que documente todos los endpoints: método, URL, parámetros y ejemplo de respuesta."
🔎 Análisis de repositorios públicos
"Busca en GitHub proyectos open source que usen [tecnología X] para [caso de uso Y]. Dame los 5 más relevantes con una descripción de su arquitectura."
